    A light-weight PHP image resizer with dynamic caching.

    A simple solution to fulfil the most common image resizing tasks with GD.

    Features

    • Supported Image Types
      • JPEG
      • GIF + Transparency
      • PNG + Transparency
    • Resize
      • Standard/Proportional
      • Crop-to-fit
      • Letterbox
      • Percentage
    • Manipulation
      • Sharpen
      • Rotate
      • Aspect Ratio toggle
      • Contrain dimensions
      • Quality/Compression
    • File Saving
      • Custom destination path
      • Automated cache with custom expiration setting

    Example 1:

    Resize and save an image to a specfic location.

    try {
    include_once("imagelite.class.php");

    $srcUri = ImageLite::inst("./src.jpg")->sharpen(7)->resize(100,200)->save("./new.jpg")->getUri();

    echo "<img src='{$srcUri}' />";
}
catch (Exception $e) {
    echo $e->getMessage();
}

    Example 2:

    Resize an image and store it in the cache.

    try {
    include_once("imagelite.class.php");

    ImageLite::setCacheRootDirectory("/path/to/images/cache/directory", true);

    $img = ImageLite::inst("/path/to/src/image.jpg")->quality(70)->resize(100,200)->save();

    echo "<img src='". $img->getUri() ."' />";
}
catch (Exception $e) {
    echo $e->getMessage();
}

    Example 3:

    Resize an image and store it in the cache (note that chaining is not used).

    try {
    include_once("imagelite.class.php");

    ImageLite::setCacheRootDirectory("/path/to/cache/directory", true);
    ImageLite::setMode(0777);
    ImageLite::setCacheUri("/assets/cache/");

    $img = ImageLite::inst("/path/to/original/image.jpg");
    $img->quality(70);
    $img->aspectRatio(false);
    $img->sharpen(8);
    $img->resize(100,200);
    $img->save();
    echo "<img src='". $img->getUri() ."' />";
}
catch (Exception $e) {
    echo $e->getMessage();
}

    Set width/height for letterbox resize
    

    resizeLetterbox(integer | null $width, integer | null $height, string $bgColour, integer $bgAlpha) : object
    

    

    

    

    Expect the following for background colours:
    
JPEG: Only a solid background colour can be applied.
    
PNG: A solid & transparent background colour can be applied.
    
GIF: A solid background colour can be applied or a full transparency is used if $bgAlpha is 127
    

    

    
example
resizeLetterbox(250); 
    Resize image to match 250x250 using a letter box effect.    		

    

    
example
resizeLetterbox(null, 300); 
    Resize image to match 300x300 using a letter box effect.    		

    

    
example
resizeLetterbox(250, 300); 
    Resize image to match 250x300 using a letter box effect.    		

    

    
example
resizeLetterbox(250, 300, "FFFFFF"); 
    Resize image to match 250x300 using a letter box effect with a white background.    		

    

    
example
resizeLetterbox(250, 300, "FFFFFF", 80); 
    Resize image to match 250x300 using a letter box effect with a transparent white background.

    Get a memory exhausted message
    

    getMemoryExhaustedMsg(string $filePath) : boolean | string
    

    

    

    Based on the width/height of the source image and the current memory limit we must estimate
how much memory will be required to resize the image without exhausting the memory.
ie. Prevent an error such as "Fatal error: Allowed memory size of 76532345 bytes exhausted (tried to allocate 345 bytes) in page.php
    

    Parameters
    

    

    $filePath
    
string
    A file path can be added to the message (optional)
    

    

    Returns
    

    
booleanstringString: Memory will potentially be exhausted. False: Memory will not be exhausted.
